Authentication:

The user can sign into their GitHub Account. Checking 'remember me' will save the creditentials securely, for future authentication.


If the user has previously signed in with 'remember me' checked, sign in with Saved will allow the user use those credentials without needing to fill out the input fields.


This button allows the user to proceed to the main application without authentication. The user can use the application, provided they don't require any authentication.

Using the Repository View:

Once you have opened/cloned a repository, you should be able to see the file panel on the left.

The File panel will show any modified files in the repository.
Each modified file can be distinguished by its colour:
A Green file represents a new file to the repository.
A Red file represents a file removed from the repository.
An orange file represents a modified file from the repositroy.

Give the file panel a test run by:

1. Opening, cloning or initialising a repository.

2. Add a new file to the repository. The file should appear in green.

If the repository has existing files:

3. Modify a file in the repository. The file should appear in orange.

4. Remove a file from the repository. The file should appear in red.


Also visible in the file panel is the commit message input and commit button.
Once you have finished making changes through VisualGit, you can add a message describing the changes and 'commit' it. Each commit represents a set of changes since the previous iteration or commit.


Clicking on a modifed file from the file panel will open up the diff panel.

The Diff panel will show any modifications in the file since the last commit.
Each modified line can be distinguished by its colour:
A Green line represents the new line added to the file.
A Red line represents a line removed from the file. Give the Diff panel a test run by:

1. Opening an exisitng repository.

2. Click on a modified file to open the diff panel.

If no files appear in the file panel, you will need to manually modify the repository.

The modified lines should be highlighted based on the type of change.


Opening a repository will show a graph detailing the history of changes.

In the graph area, the history of changes are visualled for accessibility.
When zoomed in, the commits are condensed into nodes. Hovering over these will show the number of commits the nodes are made up of.
Branches from the master branch can be seen as labels with arrows pointing to the node or commit it was branched from, when created.




Using the Navigation Bar:

Once you have committed changes to the repository, you can push the changes to the remote repository, using the 'Push' button.

Note: A remote repository is the common repository shared between the team, hosted on a service such as Github. The current repository you are working on is a local version.


If the local branch is not up to date with the remote, you can update it using the 'Pull' button.
